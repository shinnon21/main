# メール送信設定手順（Google Workspace / Gmail SMTP）

お問い合わせフォーム（Contact Form 7）の送信メールを、**Google Workspace の Gmail SMTP 経由**で送るための手順。

## なぜ必要か（背景）

- WordPress は既定でサーバーの `mail()` 関数で送信するが、**GCP VM からの直送はほぼ届かない**（GCE は送信ポート25を完全ブロック／クラウドIP直送は迷惑メール判定・拒否されやすい／SPF・DKIM・逆引きが伴わない）。
- **GCP 自体に汎用のメール送信サービスはない**ため、外部SMTPを経由させる。
- `@shinnosuke-kobayashi.jp` は Google Workspace 運用（`hello@`＝本アカウント、`contact@`＝そのエイリアス）なので、**Gmail SMTP を利用**するのが最短・最良。
  - Google のサーバーを認証済みアカウントとして経由するため、**SPF/DKIM/DMARC が自動でパス**し、追加のDNS作業は原則不要。
- 送信上限は Workspace で 1日約2,000通。お問い合わせ用途では十分。

## 方式の選択

| 方式 | 手間 | 備考 |
|---|---|---|
| **A. WP Mail SMTP + アプリパスワード**（推奨） | 小 | `smtp.gmail.com:587`。2段階認証＋アプリパスワードだけで完了 |
| B. WP Mail SMTP + Google API（OAuth） | 中 | より堅牢だが GCP で OAuth クライアント作成が必要 |

---

## 手順A：アプリパスワード方式（推奨）

### 1. 2段階認証プロセスを有効化
- `hello@shinnosuke-kobayashi.jp` でログイン → [Googleアカウント](https://myaccount.google.com/) → **セキュリティ** → **2段階認証プロセス**を ON。
- （管理者側で無効化されている場合：管理コンソール → セキュリティ → 認証 → 2段階認証プロセス を許可）

### 2. アプリパスワードを発行
- https://myaccount.google.com/apppasswords を開く（2段階認証ON後に表示される）。
- アプリ名「WordPress」などで作成 → 表示された**16桁のパスワードをコピー**（この画面でしか表示されない）。
- ※ 管理コンソールでアプリパスワードが無効の場合：管理コンソール → セキュリティ → アクセスとデータ管理 → 「安全性の低いアプリ／アプリパスワード」の設定を確認。

### 3. WP Mail SMTP プラグインを導入・設定
- 管理画面 → プラグイン → 新規追加 → **「WP Mail SMTP」**（by WPForms）→ インストール → 有効化。
- 設定 → **WP Mail SMTP** → 一般：
  - **From Email**: `hello@shinnosuke-kobayashi.jp`（`contact@` を使う場合は下記「補足」参照）
  - **Force From Email**: ON（CF7側の設定に関わらず送信元を固定）
  - **From Name**: `小林慎之助 公式サイト` 等
  - **Force From Name**: ON
  - **Mailer**: **Other SMTP**
  - **SMTP Host**: `smtp.gmail.com`
  - **Encryption**: **TLS**
  - **SMTP Port**: `587`
  - **Auto TLS**: ON
  - **Authentication**: ON
  - **SMTP Username**: `hello@shinnosuke-kobayashi.jp`（＝アプリパスワードを発行したアカウント）
  - **SMTP Password**: 手順2でコピーした**16桁のアプリパスワード**
- 保存。

### 4. テスト送信で疎通確認
- WP Mail SMTP → **Tools（ツール）** → **Email Test** → 自分の受信できるアドレスへ送信 → 届けばSMTP設定は成功。
- エラー時は同画面のログに原因が出る（下部トラブルシュート参照）。

### 5. Contact Form 7 のメール設定を整える
- お問い合わせ → 対象フォーム → **「メール」**タブ：
  - **送信先（To）**: `contact@shinnosuke-kobayashi.jp`（問い合わせを受け取りたいアドレス）
  - **送信元（From）**: `小林慎之助 公式サイト <hello@shinnosuke-kobayashi.jp>`
    - ⚠️ フォーム入力値（`[your-name]` や `[your-email]`）を From にしない。**必ずドメインのアドレス**にする（なりすまし判定を避けるため）。
  - **追加ヘッダー**に以下を記載（送信者へそのまま返信できるようにする）:
    ```
    Reply-To: [your-email]
    ```
  - 「メール(2)」（自動返信メール）を使う場合も、From は同様にドメインアドレスにする。
- 保存。

### 6. 本番テスト
- `/contact/` から実際に送信し、次の2点を確認：
  1. フォームが**サンクス画面に切り替わる**（テーマ v1.24 実装）
  2. `contact@` に問い合わせメールが**届く**（迷惑メールフォルダも確認）

## 補足：送信元(From)を `contact@` にしたい場合

`contact@` は `hello@` のエイリアスなので、そのままでは Gmail が送信元を `hello@` に書き換える。`contact@` を差出人として使うには、`hello@` の Gmail で **「別のアドレスから送信」** を登録する：

1. `hello@` の Gmail → 設定（歯車）→ すべての設定を表示 → **アカウント** タブ。
2. 「他のメール アドレスを追加」→ `contact@shinnosuke-kobayashi.jp` を入力 → 確認メールを承認。
3. 登録後、WP Mail SMTP / CF7 の From を `contact@shinnosuke-kobayashi.jp` に変更。

登録しない場合は、From は `hello@` のままにしておく（届きさえすれば運用上は問題なし）。

## 手順B：Google API（OAuth）方式（代替・より堅牢）

アプリパスワードを使わず、OAuth 認証で送る方式。

1. WP Mail SMTP → Mailer で **「Google / Gmail」** を選択。
2. GCP コンソール（プロジェクト `shinnosuke-kbys`）で：
   - **Gmail API** を有効化。
   - **OAuth 同意画面**を設定（内部/外部）。
   - **OAuth クライアントID**（種類: ウェブアプリケーション）を作成し、**承認済みリダイレクトURI**に WP Mail SMTP 設定画面が表示する URL を登録。
3. 発行された **Client ID / Client Secret** を WP Mail SMTP に貼り付け → 「Allow / 許可」で認証。
4. From を `hello@`（または承認済みエイリアス `contact@`）に設定。
- 詳細は WP Mail SMTP 公式ドキュメント「Google mailer」を参照。

## トラブルシュート

| 症状 | 対処 |
|---|---|
| `535 Authentication failed` | 2段階認証が未有効、または通常のログインパスワードを入れている。**必ずアプリパスワード**を使う |
| Email Test は届くが本番が届かない | CF7 の From がフォーム入力値になっていないか確認（ドメインアドレスに固定）。To のアドレスが正しいか確認 |
| 迷惑メールに入る | From をドメインアドレスに固定できているか確認。Workspace の DKIM が有効か（管理コンソール → アプリ → Gmail → メールの認証 で DKIM を「開始」） |
| `SMTP AUTH` が拒否される | 管理コンソール → アプリ → Google Workspace → Gmail → エンドユーザーアクセス で「SMTP 認証済みの送信」を許可 |
| アプリパスワードの項目が出ない | 2段階認証を先に有効化する／管理コンソールでアプリパスワードが許可されているか確認 |

## 運用メモ

- アプリパスワードは漏洩時に個別失効できる。退職者アカウント等では発行元アカウントの状態に注意。
- 送信元アカウント（`hello@`）を無効化・削除するとフォーム送信が止まる。
- 大量送信（メルマガ等）を将来行う場合は SES / Brevo 等の専用サービスへ切り替えを検討（現状のお問い合わせ用途は Gmail SMTP で十分）。
