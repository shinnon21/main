## 1. プロジェクト概要

小林慎之助のポートフォリオを主軸とした個人サイトを構築する。情報設計・ページ構成・機能・導線は参考サイト「おしえて！アミタさん」を踏襲しつつ、主役コンテンツを「記事」から「実績（Works）」に置き換えて再設計する。

### 1.1 参考サイトの扱い（権利面の前提）

- **踏襲してよいもの**: 情報設計（ページ構成・ナビ構造）、機能（条件検索・ランキング等）、レイアウトパターン（カード型一覧・サイド型ランキング等）。これらはアイデア・事実に属し著作権保護の対象外。
- **流用しないもの（独自制作）**: ロゴ、写真・イラスト・アイコン・動画素材、キャッチコピーや記事等の文章、WordPressテーマのコード・CSS、配色・フォント指定のトンマナ一式。
- サイト名・ドメインも参考サイトと誤認されない独自のものを使用する。

## 5. 画面設計（ブロック構成）

各画面の詳細レイアウトは別紙ワイヤーフレーム参照。ここでは構成ブロックと仕様を定義する。

### 5.1 共通ヘッダー

4. フリーワード検索窓
5. グローバルナビ（PC: 横並び、SP: ハンバーガー展開）: 実績 ／ コラム ／ お知らせ ／ プロフィール ／ About
6. SNSアイコン（X ほか掲載アカウントは未決事項）

### 5.2 共通フッター

1. ロゴ＋サイト説明1行
2. サイトマップリンク（全主要ページ）
3. SNSアイコン
4. 資料DL・お問い合わせのCTAボタン
5. プライバシーポリシー ／ About
6. コピーライト表記

| 9 | お問い合わせCTA | フッター直前の全幅CTA |

### 5.4 実績一覧（/works/）

1. サブページ共通MV（ページ名英字ラベル＋H1）＋パンくず
2. おすすめ実績（ピックアップ2〜4件）
3. キーワード絞り込み（skillタグのチェックボックス＋適用・リセット）
4. 実績カードグリッド（サムネイル／種別バッジ／タイトル／期間／タグ）
5. ページネーション（数字＋前後送り）
6. サイド: 人気実績ランキング5件／タグクラウド

### 5.5 実績詳細（/works/{slug}/）

1. パンくず＋タイトル＋メタ（公開日・更新日・種別・タグ）
2. メインビジュアル
3. 概要テーブル: クライアント名（許諾済み表記）／業界／期間／役割／担当範囲／使用技術／公開URL
4. 本文: 背景・課題 → アプローチ → 成果（数値実績を強調表示）
5. ギャラリー（スクリーンショット複数）
6. 関連実績（同タクソノミー3件）
7. CTA: お問い合わせ＋資料DL
8. シェアボタン（X／Facebook／はてな／URLコピー）

### 5.6 コラム一覧・詳細（/column/）

- 一覧: おすすめ記事 → キーワード絞り込み → 新着リスト（公開日・更新日2軸表示）→ ページネーション。サイドに人気記事ランキング
- 詳細: タイトル＋メタ → 目次（h2自動生成）→ 本文 → 執筆者ボックス（プロフィールへ）→ 関連記事 → CTA

### 5.7 条件から探す（/searches/）

| 項目 | 仕様 |
|---|---|
| フリーワード | テキスト入力 |
| 期間 | 年月プルダウン from〜to（公開年月で絞り込み） |
| 種別 | チェックボックス: 実績／コラム／お知らせ |
| キーワード | skillタグのチェックボックス（初期は主要タグ、「すべて表示」で全展開） |
| 並び順 | 公開順／更新順の切替タブ |
| 結果 | 種別バッジ付き横型カード＋ページネーション |
| その他 | 条件リセットボタン。検索条件はGETクエリで保持（URL共有可） |

### 5.8 プロフィール（/profile/）

1. 顔写真＋氏名（ふりがな）＋肩書
2. 自己紹介文
3. 経歴タイムライン
4. スキル一覧（カテゴリ別タグ＋習熟度）
5. 資格・受賞
6. 対応可能領域（依頼メニュー）
7. SNSリンク＋お問い合わせCTA

### 5.9 資料ダウンロード（/documents/）

- Phase 1: 資料カード（サムネ＋説明）から直接DL（職務経歴書PDF等）
- Phase 2: フォーム入力後DL（リード獲得型）へ拡張可能な構造にしておく

### 5.10 お問い合わせ（/contact/）

- 項目: 氏名（必須）／会社名／メール（必須）／お問い合わせ種別（採用・案件相談・その他）／本文（必須）／プライバシーポリシー同意（必須）
- 確認画面なし・インラインバリデーション → /contact/thanks/ へ遷移（GA4コンバージョン計測点）
- スパム対策: reCAPTCHA v3

### 5.11 About（/about/）

サイトの目的・掲載内容のポリシー・引用や実績記載の注意書き・リンクポリシー・運営者情報（プロフィールへのリンク）。

### 5.12 404・検索0件

イラスト＋トップ／実績一覧／条件検索への導線。検索0件時は人気コンテンツを代替表示。

## 6. コンテンツモデル（WordPress設計）

### 6.1 カスタム投稿タイプ

| 投稿タイプ | スラッグ | 用途 | アーカイブ | 初期想定件数 |
|---|---|---|---|---|
| 実績 | works | ポートフォリオ本体 | あり | 8〜15件 |
| コラム | column | ノウハウ・制作記録 | あり | 5件〜（継続追加） |
| お知らせ | news | 更新情報・活動報告 | あり | 3件〜 |
| 資料 | document | DL資料の管理（一覧は固定ページに出力） | なし | 1〜3件 |
| 登壇・イベント | event | Phase 2 | あり | — |

標準の「投稿」は使用しない（リダイレクトまたは非表示）。

### 6.2 タクソノミー

| タクソノミー | スラッグ | 対象 | 例 |
|---|---|---|---|
| 実績種別 | works_type | works | Web制作／UIデザイン／システム開発／ディレクション |
| スキルタグ | skill | works, column, news 横断 | #WordPress #Next.js #Figma #SEO #採用サイト |
| 業界 | industry | works | 人材／SaaS／EC／メディア |

skillタグを全投稿タイプ共通にすることで、条件検索・ティッカー・関連コンテンツを横断で機能させる（参考サイトのタグ設計を踏襲）。

### 6.3 カスタムフィールド（ACF）

**works**

| フィールド | 型 | 備考 |
|---|---|---|
| client_name | text | 掲載許諾済みの表記。非公開案件は「非公開（業界名）」 |
| period_start / period_end | date | 期間表示用 |
| role | text | 役割（例: デザイン＋実装） |
| scope | checkbox | 担当範囲（要件定義／設計／デザイン／実装／運用） |
| tech_stack | text | 使用技術 |
| site_url | url | 公開URL（任意） |
| gallery | gallery | スクリーンショット |
| kpi_results | repeater | 数値成果（ラベル＋値） |
| is_featured | true/false | トップ注目実績に表示 |

**document**: file（PDF）／thumbnail／description。
**column / news**: 標準フィールドで足りる想定。更新日は modified を利用し、公開日と2軸表示（参考サイト踏襲）。

## 8. デザイン方針（ブランドガイド確定版）

支給されたブランドガイド（brand-assets/ 保管）に基づくクリムゾン基調。参考サイトの配色・ロゴ・素材・CSSは不使用。テーマv1.1に反映済み。

| トークン | 値 | 用途 |
|---|---|---|
| --crimson | #C22740 | プライマリ（ロゴ・ボタン・アクセント） |
| --crimson-dark | #A61F35 | hover |
| --deep | #84192A | 押下・強調・濃色帯（ティッカー／DL帯／バッジ） |
| --tint | #F6D8DD | タグ・強調背景 |
| --tint-light | #FBECEE | セクション背景 |
| --bg | #FDF9F9 | ページ背景 |
| --ink | #1A1A1A | 本文テキスト |
| --gray | #5C5C5C | 補足テキスト |
| --line | #E6DCDE | 罫線・区切り |

- ロゴ: 横組み（ヘッダー使用）／マーク（ファビコン用）／プライマリの3種SVG＋PNGをテーマ assets/img/ に同梱
- コントラスト: #C22740 on #FFF ≈ 5.7:1（WCAG AA適合）。小さい文字でクリムゾンを使う場合は --deep を使用（ブランドガイド注記に準拠）
- フォント: Noto Sans JP（本文）＋ Inter（英字ラベル・数字）
- レイアウトパターン: カード型一覧／サイド型ランキング／サブページ共通MV＋パンくず／タグチップ
- レスポンシブ: モバイルファースト。ブレークポイント 600px／960px
- ヒーローは動画を使わず、静止画＋軽量アニメーションで表示速度を優先
