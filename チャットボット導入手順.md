# AIチャットボット導入手順（Vertex AI / Gemini）

サイト右下に表示するAIチャットボットを本番で有効化するための手順。テーマ側の実装（v1.20〜）は自動デプロイで反映されるが、**GCP側の準備（本手順）と管理画面での有効化が必要**。

## 仕組みの概要

- ウィジェット（`assets/chatbot.js`）→ WordPress REST API（`/wp-json/kobayashi/v1/chat`）→ Vertex AI `generateContent` の順で呼び出す。APIキーやトークンがブラウザに出ることはない
- **創作（ハルシネーション）対策**: プロフィール・実績・コラム・お知らせ・About等の掲載情報を全てシステムプロンプトに注入し、「掲載情報のみを根拠に回答。ない情報は答えず、お問い合わせフォームへ誘導」を厳命。temperature=0。掲載情報は6時間キャッシュされ、記事更新で自動的に再生成される
- **認証**: VM（GCE）のサービスアカウントを利用（メタデータサーバーからトークン取得。鍵ファイル・プラグイン不要）
- **コスト防御**: レート制限（IP毎 8回/分・40回/日、サイト全体 800回/日）＋回答は最大1,024トークン

## モデルとコスト

| モデル | 入力/100万tok | 出力/100万tok | 備考 |
|---|---|---|---|
| `gemini-2.5-flash-lite`（既定） | $0.10 | $0.40 | 品質と価格のバランス◎ |
| `gemini-2.0-flash-lite` | $0.075 | $0.30 | 最安。旧世代 |

1回の質問は入力1〜2万トークン（掲載情報の注入分）＋出力数百トークン ≒ **約0.2〜0.4円**。サイト全体の上限800回/日でも最大約300円/日で頭打ち。モデルは管理画面から変更可能（コード変更不要）。

## 手順1: Vertex AI APIの有効化

> **注意**: gcloud操作は必ず `--account=hello@shinnosuke-kobayashi.jp` で行う（weeave.comの権限は削除済み）。

```bash
gcloud services enable aiplatform.googleapis.com \
  --project=shinnosuke-kbys --account=hello@shinnosuke-kobayashi.jp
```

## 手順2: VMサービスアカウントに権限付与

VMが使っているサービスアカウントを確認し、Vertex AIユーザーロールを付与する。

```bash
# サービスアカウントの確認（通常は <プロジェクト番号>-compute@developer.gserviceaccount.com）
gcloud compute instances describe wp-portfolio --zone=asia-northeast1-b \
  --project=shinnosuke-kbys --account=hello@shinnosuke-kobayashi.jp \
  --format="value(serviceAccounts[].email)"

# ロール付与（<SA_EMAIL> を上の結果に置き換える）
gcloud projects add-iam-policy-binding shinnosuke-kbys \
  --member="serviceAccount:<SA_EMAIL>" --role="roles/aiplatform.user" \
  --account=hello@shinnosuke-kobayashi.jp
```

## 手順3: VMのアクセススコープ変更（要・停止→起動）

GCEのデフォルトスコープにはVertex AIが含まれないため、`cloud-platform` スコープに変更する。**VMの停止→起動が必要（ダウンタイム約1〜2分）**。静的IP（35.189.147.122）は予約済みのため変わらない。Apache・certbotは起動時に自動復帰する。

```bash
gcloud compute instances stop wp-portfolio --zone=asia-northeast1-b \
  --project=shinnosuke-kbys --account=hello@shinnosuke-kobayashi.jp

gcloud compute instances set-service-account wp-portfolio --zone=asia-northeast1-b \
  --scopes=cloud-platform \
  --project=shinnosuke-kbys --account=hello@shinnosuke-kobayashi.jp

gcloud compute instances start wp-portfolio --zone=asia-northeast1-b \
  --project=shinnosuke-kbys --account=hello@shinnosuke-kobayashi.jp
```

> 現在のスコープは `gcloud compute instances describe wp-portfolio --zone=asia-northeast1-b --format="value(serviceAccounts[].scopes)" ...` で確認できる。既に `cloud-platform` が入っていればこの手順は不要。
>
> **代替手段**: VMを止めたくない場合は、[Vertex AI Studio](https://console.cloud.google.com/vertex-ai/studio)で発行できるAPIキー（Express）を管理画面の「APIキー」欄に入れれば手順3なしでも動く。APIキー使用時はグローバルエンドポイント（`https://aiplatform.googleapis.com/v1/publishers/google/models/…`）固定となり、ロケーション設定は無視される。キーはDBに保存されるため、恒久運用はサービスアカウント方式を推奨。

## 手順4: WordPress管理画面で有効化

1. **設定 → AIチャットボット** を開く
2. 設定値を確認（既定で入力済み: プロジェクト `shinnosuke-kbys`／ロケーション `global`／モデル `gemini-2.5-flash-lite`）→「変更を保存」
3. **「接続テストを実行」**を押し、「接続テスト成功」が出ることを確認
4. 「チャットボットを表示」にチェック →「変更を保存」
5. サイトを開き、右下のチャットボタンから動作確認（日本語版・`/en/` 英語版の両方）

## トラブルシューティング

| 症状 | 原因と対処 |
|---|---|
| 接続テスト失敗（auth） | メタデータサーバーに到達できない＝非GCE環境（ローカル検証等）で実行している。ローカル検証はAPIキーを設定する |
| 接続テスト失敗（perm） | Vertex AIが401/403で拒否＝手順2のロール未付与、または手順3のスコープ変更が未実施（トークン自体はスコープ不足でも発行される点に注意）。`curl -H "Metadata-Flavor: Google" "http://metadata.google.internal/computeMetadata/v1/instance/service-accounts/default/scopes"` で `cloud-platform` が出るか確認 |
| 接続テスト失敗（api） | 手順1のAPI未有効化、モデル名の打ち間違い等。詳細はVMのPHPエラーログ（`/var/log/apache2/error.log`）に `kb_chatbot:` で出力 |
| 接続テスト失敗（quota）／ログに Quota exceeded | Vertex AI側のクォータ（チャットAPIからはHTTP 502・`quota`で返る）。[割り当て](https://console.cloud.google.com/iam-admin/quotas)を確認、またはモデルを変更 |
| チャットに「ご利用が集中しています」（HTTP 429） | サイト内蔵のレート制限（IP毎 8回/分・40回/日、サイト全体 800回/日）。上限を変える場合は `inc/chatbot.php` の `kb_chatbot_hit()` 呼び出しの引数を調整 |
| 回答が古い | 掲載情報キャッシュは6時間（日英別）。記事を保存すれば即時再生成される |

## 運用メモ

- チャットの会話内容はサーバーに保存しない（ブラウザのタブ内のみ）
- 掲載情報にない質問には「公開されている情報の中にはございません」と答え、お問い合わせフォームへ誘導する設計（創作しない制約）
- 無効化はいつでも管理画面のチェックを外すだけ（GCP側はそのままでよい）
